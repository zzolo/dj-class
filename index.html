Data vs. Visual

The MinnData team focuses more on visual work, not necessarily data work. This means that we want to create visual narratives, and most of the time we use data for this. But there are plenty of data driven stories that don't have or require visual aspects, and plenty of visual pieces that aren't data driven.

Process

Our process loosely follows this, but there's lots of ebb and flow.

  • Idea. This can come from many places such as a reporter, an editor, a dataset, or us.
  • Plan. Analyze data, draw, or prototype with or without code.
  • Review. Review plan with editors.
  • Build. Put it all together and make it work on the web.
  • Test. Make sure it works for everyone.
  • Publish. Turn the switch.

Brides

  • Story
  • Code
  • Data requested from the MN DOT.
  • Understanding bridge performance measures.
  • "The original data supplied to MinnPost by MnDOT that was used to create the maps in this story contained errors. Changes in the standard MnDOT uses to calculate bridge load ratings caused some bridges to have lower structural condition ratings on the maps. We've since updated the maps with new data from MnDOT. The new data produced changed structural ratings for 112 of the 3,844 bridges appearing on the original map."
